# Deploying to Render

This guide walks through deploying the Helpdesk Ticket System to [Render](https://render.com) using the included `render.yaml` Blueprint file. The Blueprint provisions a Python web service and a free PostgreSQL database in one step.

---

## Prerequisites

- The repository must be pushed to GitHub (or GitLab).
- A Render account — sign up free at [render.com](https://render.com).
- **Before deploying**, decide on an `ADMIN_REGISTRATION_CODE`. This is the secret your admins will type when creating their accounts. Keep it offline or in a password manager — it is never committed to the repository.

---

## Method A — Blueprint deploy via render.yaml (recommended)

This approach uses `render.yaml` to provision everything automatically.

### Step 1 — Push the repository to GitHub

```bash
git add .
git commit -m "Add Render deployment files"
git push origin main
```

### Step 2 — Create a new Blueprint on Render

1. Log in to Render → click **New** → **Blueprint**.
2. Connect your GitHub account if prompted.
3. Select your repository.
4. Render reads `render.yaml` and shows a preview of the resources it will create:
   - One **Web Service** (`helpdesk-ticket-system`)
   - One **PostgreSQL database** (`helpdesk-db`)
5. Click **Apply**.

### Step 3 — Set the ADMIN_REGISTRATION_CODE

The `render.yaml` marks `ADMIN_REGISTRATION_CODE` as `sync: false` — Render will not read it from the file and you must set it before the service starts:

1. Go to your **Web Service** → **Environment** tab.
2. Find `ADMIN_REGISTRATION_CODE` in the list.
3. Enter your chosen secret value.
4. Click **Save Changes** — Render will redeploy automatically.

> `SECRET_KEY` is generated automatically by Render (`generateValue: true`). You do not need to set it.

### Step 4 — Wait for the deploy to finish

Render will:
1. Install Python dependencies (`pip install -r requirements.txt`)
2. Run `flask db upgrade` (pre-deploy command — creates all tables)
3. Start Gunicorn (`gunicorn "app:create_app()"`)

Watch the deploy log in the Render dashboard. A successful deploy ends with:

```
[INFO] Listening at: http://0.0.0.0:10000
```

## Method B — Manual dashboard setup (no render.yaml)

If you prefer to configure everything through the Render UI:

1. **New → Web Service** → connect your GitHub repo.
2. Set **Runtime** to `Python 3`.
3. Set **Build Command**:
   ```
   pip install -r requirements.txt
   ```
4. Set **Start Command**:
   ```
   gunicorn "app:create_app()"
   ```
5. Under **Environment Variables**, add:

   | Key | Value |
   |---|---|
   | `FLASK_APP` | `app` |
   | `SECRET_KEY` | *(generate: click "Generate" in the Render UI)* |
   | `ADMIN_REGISTRATION_CODE` | *(your chosen secret)* |
   | `SESSION_COOKIE_SECURE` | `true` |
   | `FLASK_DEBUG` | `0` |
   | `DATABASE_URL` | *(see step 6)* |

6. **Create a PostgreSQL database** separately: **New → PostgreSQL** → name it `helpdesk-db` → free plan → **Create**. Once created, copy the **Internal Database URL** and paste it as the `DATABASE_URL` environment variable in your web service.

7. Click **Deploy**.

8. After the first deploy completes, open the **Shell** tab on your web service and run:
   ```bash
   flask db upgrade
   ```

## Post-deployment: seed categories

After the database is initialised, run the category seeder via the Render shell:

1. Go to your Web Service → **Shell** tab.
2. Run:
   ```bash
   python init_categories.py
   ```
3. Expected output:
   ```
   Default categories added (if not present).
   ```

This adds: Software, Hardware, Network, Account, Other.

## Environment variable reference

| Variable | Where set | Required | Description |
|---|---|---|---|
| `FLASK_APP` | render.yaml / dashboard | Yes | Tells `flask db upgrade` where the app is |
| `SECRET_KEY` | Auto-generated by Render | **Yes** | Flask session signing key — must be long and random |
| `ADMIN_REGISTRATION_CODE` | Dashboard only | Recommended | Secret code required to register admin accounts. Leave empty to disable admin self-registration. |
| `SESSION_COOKIE_SECURE` | render.yaml | Yes (on Render) | `true` — Render serves over HTTPS, so session cookies must be HTTPS-only |
| `FLASK_DEBUG` | render.yaml | Yes | `0` — debugger must be off in production |
| `DATABASE_URL` | Injected from linked DB | Yes | PostgreSQL connection string — set automatically when using render.yaml |

## Database notes

### Why PostgreSQL and not SQLite?

Render's filesystem is **ephemeral** — any file written to disk (including `support.db`) is lost when the container restarts or redeploys. PostgreSQL is a managed, persistent service.

SQLAlchemy uses the driver appropriate for the URL scheme automatically:
- `sqlite:///...` → SQLite (local dev)
- `postgresql://...` → PostgreSQL (production)

No application code changes are needed.

### Migrations

`flask db upgrade` is run automatically as a `preDeployCommand` (render.yaml method) before each new version goes live. For the manual method, run it once from the Shell after the first deploy.

### Free PostgreSQL expiry

Render's free PostgreSQL databases **expire after 90 days**. Back up your data before expiry or upgrade to a paid plan for a persistent database.

## Verification checklist

After deployment, test the following in the browser using your Render URL (e.g. `https://helpdesk-ticket-system.onrender.com`):

| Test | Expected result |
|---|---|
| Visit `/` | Home page loads with Login and Register buttons |
| Register a User account | Redirected to User Login page with success flash |
| Log in as User | User Dashboard shown |
| Navigate to `/admin/dashboard` while logged in as User | 403 Access Denied page |
| Navigate to `/this-does-not-exist` | 404 Page Not Found page |
| Register an Admin account | Requires `ADMIN_REGISTRATION_CODE` to succeed |
| Log in as Admin | Admin Dashboard shown with all tickets visible |
| Check response headers via `curl -I https://your-url.onrender.com/` | `X-Frame-Options: DENY`, `X-Content-Type-Options: nosniff`, `Content-Security-Policy` all present |
| Check Render logs | `[AUTH]`, `[TICKET]` events visible in the Render log stream |

## Common issues

| Symptom | Likely cause | Fix |
|---|---|---|
| `flask db upgrade` fails in pre-deploy | `FLASK_APP` env var not set | Add `FLASK_APP=app` in Environment tab |
| App starts but database tables missing | Migrations never ran | Open Shell → `flask db upgrade` |
| `OperationalError: no such table` | Same as above | Same fix |
| "Admin registration is disabled" | `ADMIN_REGISTRATION_CODE` is empty | Set it in Environment tab and redeploy |
| Static files / Bootstrap not loading | CSP issue | Check browser console; Bootstrap loads from `cdn.jsdelivr.net` which is whitelisted |
| App shows Render's default error page | Gunicorn failed to start | Check the deploy log for a Python import error |
